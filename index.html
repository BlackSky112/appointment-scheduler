Setup Instructions

  1. Create a Firebase Project:

    Go to the Firebase Console and create a new project. Follow the on-screen instructions.

  2. Enable Firestore:

    In your new Firebase project, navigate to the "Build" menu and select "Firestore Database." Click "Create database" and choose to start in "Test mode" for now. Select a location and click "Enable."

  3. Configure Security Rules:

    In the Firestore section, go to the "Rules" tab. Replace the default rules with the following code to allow authenticated users to read and write to the appointments collection:

    rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /artifacts/{appId}/users/{userId}/{document=**} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}

    Click "Publish" to save the rules.

  4. Copy Configuration:

    In your Firebase project, go to "Project settings" (the gear icon next to "Project Overview"). Under the "General" tab, scroll down to the "Your apps" section and click the "" icon to add a web app. Register the app and copy the `firebaseConfig` object provided. You will need to insert this into a global variable for this code to work.

Once you've completed these steps, this tool will be fully functional and your appointment data will be securely stored in your personal Firebase database.
